# CS 255 Portfolio Reflection – DriverPass Project

## DriverPass Project Summary

The DriverPass project focused on designing a system for DriverPass, a company that wanted to help students prepare for their driver's license exams. The client wanted a web-based system where students could register for accounts, schedule driving lessons, take practice tests, and track their progress. The system also needed to allow administrators and employees to manage appointments, customer information, and system updates while providing secure access for different types of users.

## What I Did Well

One of the strongest parts of my work was gathering and organizing the client's requirements. I carefully reviewed the interview transcript and translated the customer's requests into clear business requirements and system design features. I also created diagrams and documentation that showed how different users would interact with the system, making the design easier to understand.

## What I Would Revise

If I could revise one part of my work, I would improve the system design document by adding more detail to the UML diagrams and including additional explanations for how the different system components communicate with one another. While the design met the project requirements, providing more technical detail would make it even easier for developers to implement the system.

## Interpreting User Needs

I interpreted the user's needs by carefully reviewing the client's interview and identifying the features that were most important to the business. I ensured those requirements were reflected throughout the business requirements document and system design. Considering user needs is critical because the success of any system depends on whether it solves the customer's problems and provides an easy, reliable experience for the people who will actually use it.

## My Approach to Software Design

When designing software, I begin by understanding the client's goals and gathering detailed requirements before creating any technical designs. From there, I organize the information into models, diagrams, and documentation that describe how the system should function. In future projects, I will continue using techniques such as requirements analysis, UML diagrams, use case modeling, and iterative design because they help identify problems early and ensure the final system meets both business and user needs.
